每次从字符串里拿出一个字符和模式中的字符去匹配。先来分析如何匹配一个字符。如果模式中的字符ch是'.'，那么它可以匹配字符串中的任意字符。如果模式中的字符ch不是'.'，而且字符串中的字符也是ch，那么它们也匹配。当字符串中的字符和模式中的字符相匹配时，接着匹配后面的字符。

相对而言,当模式中的第二个字符不是'*'时，问题要简单很多。如果字符串中的第一个字符和模式中的第一个字符相匹配，那么在字符串和模式上都向后移动一个字符，然后匹配剩余的字符串和模式。如果字符串中的第一个字符和模式中的第一个字符不相匹配，则直接返回 false。

当模式中的第二个字符是 '*' 时，问题要复杂一些，因为可能有多种不同的匹配方式。一种选择是在模式上向后移动两个字符。这相当于和它前面的字符被忽略了，因为'*'可以匹配字符串中的0个字符。如果模式中的第一个字符和字符串中的第一个字符相匹配，则在字符串上向后移动一个字符，而在模式上有两种选择：可以在模式上向后移动两个字符，也可以保持模式不变。

分析

题目要求求出给定数组中的众数，也就是出现次数大于$\frac{n}{2}$的数。我刚开始想到的解法是用一个字典把数组中数字出现的次数都保存下来，但是这样消耗了太多的空间。后来了解到有一个算法叫做Boyer–Moore majority vote algorithm)，它的步骤如下：

  • 初始化一个候选元素 candidate 和 计数器 counter = 0
  • 对数组中每一个元素 x：
    • 如果 counter = 0，那么 candidate = x
    • 如果 candidate = x，counter += 1，否则 counter -= 1
  • candidate 即为所求

感觉直观上不是很容易想出来，举个例子就容易看明白了：就比如说数组「1 3 2 3 3」，第一次循环 candidate = 1， counter = 1；第二次循环 candidate = 1， counter = 0；第三次循环 candidate = 2， counter = 1；第四次循环 candidate = 2， counter = 0；最后一次循环循环 candidate = 3， counter = 1；最终 candidate = 3。我最后是这么理解的，在扫描数组的过程中，肯定会遇见真正的众数，而众数的数量是大于数组长度的一半的，所以最终的 candidate 肯定为众数并且 counter 就等于众数的个数减去非众数的个数。

快速排序

发表于 2019-08-05 | 分类于 leetcode

快速排序又称划分交换排序，简称快排，它与归并排序算法一样，也是分治策略的典型应用，但二者之间也有本质区别。归并排序的计算量主要消耗于有序子向量的归并操作，而子向量的划分几乎不消耗时间（就是对半分）；快排正好相反，它可以在$O(1)$时间内，由子问题的解直接得到原问题的解，但为了将原问题划分为两个子问题，却需要$O(n)$的时间（选取轴点）。

所谓轴点（pivot），就是左侧元素均不比它大，右侧元素均不比它小的点。如图所示：

快排的思路就是，先选取一个轴点（轴点选取好之后它就不会再变动了，即这个位置上的元素就是最终排好序的数组中该位置上的元素，可以看成这个点已经是排好序的了），然后把这个数组分为轴点左右两边两个子数组，再分别对它们进行快排，注意到最终排序好的数组中轴点左/右边的成员和当前可能无序的左/右子数组中的成员是相同的，即现在轴点右边的元素不会跑到轴点左边去，左边的也不会跑到右边去。所以这两个子数组的排序可以独立进行，更重要的是，一旦它们都完成排序，即可立即（在$O(1)$时间内）得到整个数组的排序结果。排序的代码为：
