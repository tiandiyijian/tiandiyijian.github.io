题目描述

Given a linked list, return the node where the cycle begins. If there is no cycle, return null.

To represent a cycle in the given linked list, we use an integer pos which represents the position (0-indexed) in the linked list where tail connects to. If pos is -1, then there is no cycle in the linked list.

分析

这题和环形链表很像，只不过那道题只需要判断是否有环，而这道题还要寻找环开始的位置，而且同样是不使用额外的空间。我想了半天也想不出什么好办法，最后看了答案才知道这道题就跟高中物理题似的，具体看图吧：

这是一张不太好看的图，s代表从头到环开始位置的距离，m代表从环开始位置到快慢指针相遇的位置的距离，l代表环的周长，假设慢指针走的距离为k，那么快指针就是2k。我先来证明当慢指针从环开始的位置（设为点S）到两指针相遇的位置（设为P）走的距离不到l，也就是说慢指针还没走一圈快指针就追上它了。

假设慢指针到达S的时候快指针到S的距离为x(x≥0)，从S点到P点慢指针走的距离为d，那么快指针就是2d，那么

d是小于等于l的，这就说明了慢指针还没有走一圈就和快指针相遇了，那么

这说明什么，这说明从P点继续向前走到S点的距离也是s啊，我们让慢指针回到头结点，然后再让这两个指针一次一步地向前走，最终这两个指针会在S点相遇，这个点就是我们要求的点。
